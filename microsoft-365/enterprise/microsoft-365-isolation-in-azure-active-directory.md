<span data-ttu-id="7bd30-104">Azure Active Directory (Azure AD) è stato progettato per ospitare più tenant in modo estremamente sicuro tramite l'isolamento dei dati logici.</span><span class="sxs-lookup"><span data-stu-id="7bd30-104">Azure Active Directory (Azure AD) was designed to host multiple tenants in a highly secure way through logical data isolation.</span></span> <span data-ttu-id="7bd30-105">L'accesso ad Azure AD è limitato da un livello di autorizzazione.</span><span class="sxs-lookup"><span data-stu-id="7bd30-105">Access to Azure AD is gated by an authorization layer.</span></span> <span data-ttu-id="7bd30-106">Azure AD isola i clienti che usano i contenitori tenant come limiti di sicurezza per salvaguardare il contenuto di un cliente in modo che non possa essere accessibile o compromesso dai co-tenant.</span><span class="sxs-lookup"><span data-stu-id="7bd30-106">Azure AD isolates customers using tenant containers as security boundaries to safeguard a customer's content so that the content cannot be accessed or compromised by co-tenants.</span></span> <span data-ttu-id="7bd30-107">Tre controlli vengono eseguiti dal livello di autorizzazione di Azure AD:</span><span class="sxs-lookup"><span data-stu-id="7bd30-107">Three checks are performed by Azure AD's authorization layer:</span></span>

- <span data-ttu-id="7bd30-108">L'entità è abilitata per l'accesso al tenant di Azure AD?</span><span class="sxs-lookup"><span data-stu-id="7bd30-108">Is the principal enabled for access to Azure AD tenant?</span></span>
- <span data-ttu-id="7bd30-109">L'entità è abilitata per l'accesso ai dati in questo tenant?</span><span class="sxs-lookup"><span data-stu-id="7bd30-109">Is the principal enabled for access to data in this tenant?</span></span>
- <span data-ttu-id="7bd30-110">Il ruolo dell'entità in questo tenant è autorizzato per il tipo di accesso ai dati richiesto?</span><span class="sxs-lookup"><span data-stu-id="7bd30-110">Is the principal's role in this tenant authorized for the type of data access requested?</span></span>

<span data-ttu-id="7bd30-111">Nessun applicazione, utente, server o servizio può accedere ad Azure AD senza l'autenticazione e il token o il certificato adeguati.</span><span class="sxs-lookup"><span data-stu-id="7bd30-111">No application, user, server, or service can access Azure AD without the proper authentication and token or certificate.</span></span> <span data-ttu-id="7bd30-112">Le richieste vengono rifiutate se non sono accompagnate da credenziali appropriate.</span><span class="sxs-lookup"><span data-stu-id="7bd30-112">Requests are rejected if they are not accompanied by proper credentials.</span></span>

<span data-ttu-id="7bd30-113">In effetti, Azure AD ospita ogni tenant nel proprio contenitore protetto, con criteri e autorizzazioni per e all'interno del contenitore esclusivamente di proprietà e gestito dal tenant.</span><span class="sxs-lookup"><span data-stu-id="7bd30-113">Effectively, Azure AD hosts each tenant in its own protected container, with policies and permissions to and within the container solely owned and managed by the tenant.</span></span>
 
![Contenitore di Azure](../media/office-365-isolation-azure-container.png)

<span data-ttu-id="7bd30-115">Il concetto di contenitori tenant è molto inciso nel servizio directory a tutti i livelli, dai portali fino all'archiviazione permanente.</span><span class="sxs-lookup"><span data-stu-id="7bd30-115">The concept of tenant containers is deeply ingrained in the directory service at all layers, from portals all the way to persistent storage.</span></span> <span data-ttu-id="7bd30-116">Anche quando più metadati del tenant di Azure AD sono archiviati nello stesso disco fisico, non esiste alcuna relazione tra i contenitori oltre a quella definita dal servizio directory, che a sua volta è dettata dall'amministratore tenant.</span><span class="sxs-lookup"><span data-stu-id="7bd30-116">Even when multiple Azure AD tenant metadata is stored on the same physical disk, there is no relationship between the containers other than what is defined by the directory service, which in turn is dictated by the tenant administrator.</span></span> <span data-ttu-id="7bd30-117">Non possono esserci connessioni dirette all'archiviazione di Azure AD da qualsiasi applicazione o servizio richiedente senza prima passare attraverso il livello di autorizzazione.</span><span class="sxs-lookup"><span data-stu-id="7bd30-117">There can be no direct connections to Azure AD storage from any requesting application or service without first going through the authorization layer.</span></span>

<span data-ttu-id="7bd30-118">Nell'esempio seguente, Contoso e Fabrikam dispongono di contenitori separati e dedicati e, anche se tali contenitori possono condividere parte della stessa infrastruttura sottostante, ad esempio server e archiviazione, rimangono separati e isolati l'uno dall'altro e sono recinti da livelli di autorizzazione e controllo dell'accesso.</span><span class="sxs-lookup"><span data-stu-id="7bd30-118">In the example below, Contoso and Fabrikam both have separate, dedicated containers, and even though those containers may share some of the same underlying infrastructure, such as servers and storage, they remain separate and isolated from each other, and gated by layers of authorization and access control.</span></span>
 
![Contenitori dedicati di Azure](../media/office-365-isolation-azure-dedicated-containers.png)

<span data-ttu-id="7bd30-120">Inoltre, non ci sono componenti dell'applicazione che possono essere eseguiti da Azure AD e non è possibile che un tenant viri forzatamente l'integrità di un altro tenant, accedono alle chiavi di crittografia di un altro tenant o leggono i dati non elaborati dal server.</span><span class="sxs-lookup"><span data-stu-id="7bd30-120">In addition, there are no application components that can execute from within Azure AD, and it is not possible for one tenant to forcibly breach the integrity of another tenant, access encryption keys of another tenant, or read raw data from the server.</span></span>

<span data-ttu-id="7bd30-121">Per impostazione predefinita, Azure AD non consente tutte le operazioni emesse da identità in altri tenant.</span><span class="sxs-lookup"><span data-stu-id="7bd30-121">By default, Azure AD disallows all operations issued by identities in other tenants.</span></span> <span data-ttu-id="7bd30-122">Ogni tenant è isolato logicamente all'interno di Azure AD tramite controlli di accesso basati sulle attestazioni.</span><span class="sxs-lookup"><span data-stu-id="7bd30-122">Each tenant is logically isolated within Azure AD through claims-based access controls.</span></span> <span data-ttu-id="7bd30-123">Le operazioni di lettura e scrittura dei dati della directory hanno come ambito i contenitori tenant e sono recinti da un livello di astrazione interno e da un livello di controllo dell'accesso basato sui ruoli (RBAC, Role-Based Access Control), che insieme applicano il tenant come limite di sicurezza.</span><span class="sxs-lookup"><span data-stu-id="7bd30-123">Reads and writes of directory data are scoped to tenant containers, and gated by an internal abstraction layer and a role-based access control (RBAC) layer, which together enforce the tenant as the security boundary.</span></span> <span data-ttu-id="7bd30-124">Ogni richiesta di accesso ai dati della directory viene elaborata da questi livelli e ogni richiesta di accesso in Microsoft 365 viene impostata in base alla logica precedente.</span><span class="sxs-lookup"><span data-stu-id="7bd30-124">Every directory data access request is processed by these layers and every access request in Microsoft 365 is policed by the logic above.</span></span>

<span data-ttu-id="7bd30-125">Azure AD dispone di partizioni Nord America, U.S. Government, Unione Europea, Germania e World Wide.</span><span class="sxs-lookup"><span data-stu-id="7bd30-125">Azure AD has North America, U.S. Government, European Union, Germany, and World Wide partitions.</span></span> <span data-ttu-id="7bd30-126">Un tenant esiste in una singola partizione e le partizioni possono contenere più tenant.</span><span class="sxs-lookup"><span data-stu-id="7bd30-126">A tenant exists in a single partition, and partitions can contain multiple tenants.</span></span> <span data-ttu-id="7bd30-127">Le informazioni sulla partizione vengono astratte dagli utenti.</span><span class="sxs-lookup"><span data-stu-id="7bd30-127">Partition information is abstracted away from users.</span></span> <span data-ttu-id="7bd30-128">Una determinata partizione (inclusi tutti i tenant al suo interno) viene replicata in più datacenter.</span><span class="sxs-lookup"><span data-stu-id="7bd30-128">A given partition (including all the tenants within it) is replicated to multiple datacenters.</span></span> <span data-ttu-id="7bd30-129">La partizione per un tenant viene scelta in base alle proprietà del tenant (ad esempio, il codice paese).</span><span class="sxs-lookup"><span data-stu-id="7bd30-129">The partition for a tenant is chosen based on properties of the tenant (e.g., the country code).</span></span> <span data-ttu-id="7bd30-130">I segreti e altre informazioni riservate in ogni partizione vengono crittografati con una chiave dedicata.</span><span class="sxs-lookup"><span data-stu-id="7bd30-130">Secrets and other sensitive information in each partition is encrypted with a dedicated key.</span></span> <span data-ttu-id="7bd30-131">Le chiavi vengono generate automaticamente quando viene creata una nuova partizione.</span><span class="sxs-lookup"><span data-stu-id="7bd30-131">The keys are generated automatically when a new partition is created.</span></span>

<span data-ttu-id="7bd30-132">Le funzionalità del sistema Azure AD sono un'istanza univoca di ogni sessione utente.</span><span class="sxs-lookup"><span data-stu-id="7bd30-132">Azure AD system functionalities are a unique instance to each user session.</span></span> <span data-ttu-id="7bd30-133">Inoltre, Azure AD usa le tecnologie di crittografia per fornire l'isolamento delle risorse di sistema condivise a livello di rete per impedire il trasferimento non autorizzato e indesiderato di informazioni.</span><span class="sxs-lookup"><span data-stu-id="7bd30-133">In addition, Azure AD uses encryption technologies to provide isolation of shared system resources at the network level to prevent unauthorized and unintended transfer of information.</span></span>
