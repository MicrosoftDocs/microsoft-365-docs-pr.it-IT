## <a name="what-does-end-of-support-mean"></a>Cosa significa *fine del supporto* ?

Come la maggior parte dei prodotti Microsoft, PerformancePoint Server 2007 SP3, ProClarity software e SharePoint Server 2007 SP3, hanno un ciclo di vita di supporto, durante il quale Microsoft fornisce nuove funzionalità, correzioni di bug e aggiornamenti della sicurezza. Il ciclo di vita di un prodotto in genere si protrae per 10 anni dalla versione iniziale del prodotto. La fine del ciclo di vita è nota come fine del supporto del prodotto. Come ProClarity, PerformancePoint Server e SharePoint Server 2007 hanno raggiunto la fine del supporto, Microsoft non fornisce più:
  
- Supporto tecnico per i problemi che possono verificarsi.
    
- Correzioni di bug per i problemi individuati e che possono influire sulla stabilità e sull'usabilità dei server.
    
- Correzioni per la sicurezza per le vulnerabilità individuate e che possono rendere i server o le applicazioni vulnerabili alle violazioni della sicurezza.
    
- Aggiornamenti del fuso orario.
    
L'installazione di ProClarity, SharePoint Server 2007 SP3 e PerformancePoint Server 2007 SP3 continuerà a essere eseguita anche se il supporto è stato terminato. Tuttavia, è consigliabile eseguire la migrazione da queste applicazioni al più presto possibile.

### <a name="what-if-i-dont-upgrade-now"></a>Che cosa fare se non si esegue l'aggiornamento adesso?

È possibile scegliere di non eseguire l'aggiornamento immediatamente. I server e le applicazioni esistenti continueranno a essere eseguiti. Tuttavia, non si riceveranno ulteriori aggiornamenti, inclusi gli aggiornamenti della sicurezza, poiché il supporto è terminato. Se si verifica un problema con le applicazioni server, non sarà possibile ottenere assistenza dal supporto tecnico Microsoft.

### <a name="use-power-bi-in-the-cloud-or-on-premises"></a>Utilizzare Power BI (nel cloud o in locale)

Power BI è una famiglia di strumenti di analisi aziendale per l'analisi dei dati e la condivisione delle informazioni. Con Power BI, è possibile utilizzare origini dati locali o online per creare report e dashboard interattivi. Gli utenti possono visualizzare e utilizzare i report e i dashboard nei propri computer o dispositivi mobili.
  
Power BI non è parte di Microsoft 365 o SharePoint Server. Si tratta di un'offerta separata che include il desktop Power bi, i gateway Power BI e il servizio Power BI. Power BI si integra anche con SharePoint Online. È possibile iniziare a utilizzare Power BI gratuitamente. In base all'utilizzo dei dati e alle esigenze aziendali, è possibile eseguire l'aggiornamento in un secondo momento a Power BI Pro con Microsoft 365 E5. Per ulteriori informazioni, vedere [che cos'è Power bi?](https://go.microsoft.com/fwlink/?linkid=841341)
  
### <a name="use-reporting-services-on-premises"></a>Utilizzare Reporting Services (in locale)

SQL Server Reporting Services offre una soluzione di reporting affidabile. È possibile configurare Reporting Services in modalità nativa o in modalità integrata di SharePoint. È possibile utilizzare diversi strumenti per creare report, tra cui progettazione report, Generatore report e Power View. Con l'ultima versione di SQL Server, è inoltre possibile utilizzare Publisher di report di SQL Server Mobile per recapitare i report che vengono scalati a qualsiasi dimensione dello schermo. Questo consente agli utenti di utilizzare i report sui propri dispositivi mobili. Per ulteriori informazioni, vedere [SQL Server Reporting Services (SSRS): creare, distribuire e gestire i report per dispositivi mobili e impaginati](https://go.microsoft.com/fwlink/?linkid=841342).
  
### <a name="use-performancepoint-services-on-premises"></a>Utilizzo di PerformancePoint Services (in locale)

PerformancePoint Server 2007 è stato venduto separatamente da SharePoint Server 2007. A partire da SharePoint Server 2010, PerformancePoint Services è un'applicazione di servizio in SharePoint Server. Pertanto, non è necessario acquistare licenze o hardware server separati per l'utilizzo di PerformancePoint Services.
  
Per passare da PerformancePoint Server 2007 a PerformancePoint Services, è possibile passare a una versione più recente di SharePoint Server e configurare PerformancePoint Services. La versione di SharePoint Server da spostare per determinare se è possibile importare il contenuto del dashboard esistente da PerformancePoint Server 2007 a PerformancePoint Services.
  
- Se si esegue l'aggiornamento a SharePoint Server 2010, è possibile importare il contenuto del dashboard di PerformancePoint da PerformancePoint Server 2007 in PerformancePoint Services in SharePoint Server 2010. Per ulteriori informazioni, vedere [Import Wizard: PerformancePoint server 2007 content to SharePoint server 2010](https://go.microsoft.com/fwlink/?linkid=838873).
    
- Se si passa a SharePoint Server 2013 o SharePoint Server 2016, è molto probabile che sia necessario creare un nuovo contenuto del dashboard (origini dati, report, scorecard e pagine del dashboard).
    
Per iniziare a utilizzare il piano di aggiornamento di PerformancePoint Services, vedere le risorse seguenti:
  
- [Guida di orientamento alla fine del supporto di SharePoint Server 2007](sharepoint-2007-end-of-support.md)
    
- Quando si conosce la versione di SharePoint a cui si sta spostando, vedere l'articolo corrispondente per PerformancePoint Services:
    
  - [Pianificare PerformancePoint Services (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841363)
    
  - [Panoramica di PerformancePoint Services in SharePoint Server 2013](https://go.microsoft.com/fwlink/?linkid=841551)
    
  - [Panoramica di PerformancePoint Services in SharePoint Server 2016](https://go.microsoft.com/fwlink/?linkid=874704)
    
Quando si esegue l'aggiornamento a PerformancePoint Services, vengono apportate diverse nuove funzionalità e miglioramenti. PerformancePoint Services offre scorecard migliorate; nuove visualizzazioni, ad esempio l'albero di scomposizione e il rapporto Dettagli KPI; altri tipi di grafici; migliorare le funzionalità di filtraggio delle informazioni temporali; e la conformità alla accessibilità migliorata. Per ulteriori informazioni, vedere [What ' s New for PerformancePoint Services (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841343).
