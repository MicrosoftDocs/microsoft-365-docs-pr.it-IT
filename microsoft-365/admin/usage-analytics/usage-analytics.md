# <a name="microsoft-365-usage-analytics"></a>Analisi di utilizzo di Microsoft 365

## <a name="overview-of-microsoft-365-usage-analytics"></a>Panoramica dell'analisi dell'utilizzo di Microsoft 365

Usare Microsoft 365 analitica di utilizzo all'interno Power BI per ottenere informazioni dettagliate su come l'organizzazione adotta i vari servizi all'interno di Microsoft 365. È possibile visualizzare e analizzare i Microsoft 365 di utilizzo, creare report personalizzati e condividere le informazioni dettagliate all'interno dell'organizzazione. Puoi anche ottenere informazioni dettagliate sul modo in cui aree o reparti specifici usano Microsoft 365.
  
Microsoft 365'analisi dell'utilizzo consente di accedere a un dashboard predefinito che offre una visualizzazione tra prodotti degli ultimi 12 mesi e contiene una serie di report predefiniti. Ogni report fornisce informazioni specifiche sull'utilizzo. Le informazioni specifiche dell'utente sono disponibili per l'ultimo mese di calendario completo.
  
Il [modello di dati](usage-analytics-data-model.md) che alimenta l'app modello include gli attributi utente di Active Directory, consentendo di eseguire il pivot in determinati rapporti. Sono inclusi i seguenti attributi di Active Directory: posizione, reparto e organizzazione. 
  
Per iniziare a raccogliere dati, vedere [Abilitare l'analisi dell'utilizzo di Microsoft 365](enable-usage-analytics.md). 
  
Microsoft 365'analisi dei dati di utilizzo contiene una serie di report dettagliati nelle sezioni seguenti. 

È possibile accedere ai report dettagliati per ogni area selezionando le tabelle dati. È possibile visualizzare tutti i report predefiniti selezionando le schede nella parte inferiore del sito. Per istruzioni più dettagliate, vedere [Navigating and utilizing the reports](navigate-and-utilize-reports.md) e [Customizing the reports](customize-reports.md).

Tutti i valori del mese visualizzati nel report fanno riferimento all'ultimo mese completo. 

Questo riepilogo consente di comprendere rapidamente i modelli di utilizzo Office e come e dove collaborano i dipendenti.

## <a name="overview"></a>Panoramica

Il Microsoft 365 di panoramica contiene i report seguenti. È possibile visualizzarli scegliendo la scheda nella parte superiore della pagina del report. Tutti i valori del mese visualizzati nella sezione superiore del report fanno riferimento all'ultimo mese completo.

- **Adozione** &ndash; Offre un riepilogo riepilogativo delle tendenze di adozione. Utilizzare i report di questa sezione per informazioni su come gli utenti hanno adottato Microsoft 365, nonché su come l'utilizzo complessivo dei singoli servizi è cambiato mese dopo mese. È possibile vedere come possono essere abilitati gli utenti, quante persone nell'organizzazione usano attivamente Microsoft 365, quanti utenti stanno tornando e quanti usano il prodotto per la prima volta.

- **Utilizzo** &ndash; Offre una visualizzazione drill-down del volume di utenti attivi e delle attività chiave per ogni prodotto degli ultimi 12 mesi. Utilizzare i report di questa sezione per informazioni su come gli utenti dell'organizzazione usano Microsoft 365.

- **Comunicazione** &ndash; Puoi vedere a colpo d'occhio se gli utenti dell'organizzazione preferiscono rimanere in contatto usando Teams, Yammer, posta elettronica o Skype chiamate. È possibile osservare se vi sono cambiamenti nei modelli nell'uso degli strumenti di comunicazione tra i dipendenti. 

- **Collaborazione** &ndash; Scopri come gli utenti dell'organizzazione usano OneDrive e SharePoint per archiviare documenti e collaborare tra loro e come queste tendenze si evolvono mese dopo mese. È inoltre possibile visualizzare il numero di documenti condivisi internamente o esternamente e il numero di siti o account OneDrive di SharePoint utilizzati attivamente, suddivisi dai proprietari e da altri collaboratori.

- **Archiviazione** &ndash; Utilizzare questo report per tenere traccia dell'archiviazione cloud per cassette postali, OneDrive e SharePoint siti.

- **Mobilità** &ndash; Tenere traccia dei client e dei dispositivi che gli utenti usano per connettersi alla posta elettronica, Teams, Skype o Yammer.

Puoi anche usare i report in questa sezione per visualizzare i dispositivi in cui gli utenti hanno installato Office app. Tieni presente che per attivare un piano, un utente deve installare l'app e accedere con il proprio account.

- **Licenze** &ndash; Questo report contiene una panoramica dei tipi di licenza, del numero di utenti a cui è stato assegnato ogni tipo di licenza e della distribuzione dell'assegnazione delle licenze per ogni mese. Il valore del mese verso la parte superiore si riferisce al mese corrente e le metriche riflettono i valori aggregati dall'inizio del mese alla data corrente.

## <a name="product-usage"></a>Utilizzo del prodotto

Questo report contiene un report separato per ogni servizio Microsoft 365, inclusi Exchange, gruppi di Microsoft 365, OneDrive, SharePoint, Skype, Teams e Yammer. Ogni report contiene report utente attivi totali abilitati e totali, conteggi di entità quali cassette postali, siti, gruppi e account, nonché report sui tipi di attività, se appropriato. Tutti i valori del mese visualizzati nella sezione superiore del report fanno riferimento all'ultimo mese completo.

## <a name="user-activity"></a>Attività utente

I report attività utente sono disponibili per determinati servizi individuali. Questi report forniscono dati di utilizzo dettagliati a livello utente uniti agli attributi di Active Directory. Inoltre, il report Department Adoption consente di suddividere in base agli attributi di Active Directory in modo da poter visualizzare gli utenti attivi in tutti i singoli servizi. Tutte le metriche vengono aggregate per l'ultimo mese completo.

Sì. Per consentire a un gruppo di amministratori di collaborare con la stessa app modello, è possibile sfruttare le funzionalità dell'area di lavoro app di Power BI per ulteriori informazioni, vedere Come è consigliabile collaborare e condividere dashboard e [report?](/power-bi/collaborate-share/service-how-to-collaborate-distribute-dashboards-reports) 

### <a name="for-which-timeframe-is-data-available"></a>Per quale intervallo di tempo sono disponibili i dati?

Nella maggior parte dei report vengono visualizzati i dati relativi ai 12 mesi precedenti. Tuttavia, alcuni grafici potrebbero mostrare meno cronologia poiché la raccolta dati per prodotti e report diversi è stata avviata in momenti diversi e pertanto i dati per gli ultimi 12 mesi potrebbero non essere disponibili. Tutti i report verranno infine compilati fino a 12 mesi di cronologia. I report che mostrano i dettagli a livello di utente mostrano i dati relativi al mese completo precedente.

### <a name="what-data-is-included-in-the-template-app"></a>Quali dati sono inclusi nell'app modello?

I dati nell'app modello attualmente riguardano lo stesso set di metriche di attività disponibili nei [report attività.](../activity-reports/activity-reports.md) Quando i report vengono aggiunti ai report attività, verranno aggiunti all'app modello in una versione futura.

### <a name="how-does-the-data-in-the-template-app-differ-from-the-data-in-the-usage-reports"></a>In che modo i dati nell'app modello sono diversi dai dati nei report di utilizzo?

I dati sottostanti visualizzati nell'app modello corrispondono ai dati visualizzati nei report attività nell'Microsoft 365 di amministrazione. Le differenze principali sono che nei dati dell'interfaccia di amministrazione sono disponibili per gli ultimi 7/30/90/180 giorni, mentre l'app modello presenta i dati su base mensile per un massimo di 12 mesi.

Inoltre, i dettagli a livello utente nell'app modello sono disponibili solo per l'ultimo mese completo per gli utenti a cui è stata assegnata una licenza di prodotto ed eseguito un'attività.

### <a name="when-should-i-use-the-template-app-and-when-the-usage-reports"></a>Quando devo usare l'app modello e quando i report di utilizzo?

I [report attività sono](../activity-reports/activity-reports.md) un buon punto di partenza per comprendere l'utilizzo e l'adozione di Microsoft 365. L'app modello combina i dati di utilizzo Microsoft 365 e le informazioni di Active Directory dell'organizzazione e consente agli amministratori di analizzare il set di dati usando le funzionalità di analisi visiva di Power BI. In questo modo gli amministratori non solo visualizzano e analizzano Microsoft 365 dati di utilizzo, ma anche suddividerli in base alle proprietà di Active Directory, ad esempio reparti, posizione e così via. Possono anche creare report personalizzati e condividere le informazioni dettagliate all'interno dell'organizzazione. 

### <a name="how-often-is-the-data-refreshed"></a>Con quale frequenza vengono aggiornati i dati? 

Quando ci si connette all'app modello per la prima volta, questa verrà popolata automaticamente con i dati degli ultimi 12 mesi. Successivamente, i dati dell'app modello verranno aggiornati ogni settimana. I clienti possono scegliere di modificare la pianificazione dell'aggiornamento se l'utilizzo di questi dati richiede un ritmo di aggiornamento diverso.

Il servizio di Microsoft 365 back-end aggiorna i dati su base giornaliera e fornisce dati che sono compresi tra 5 e 8 giorni latenti dalla data corrente.

La **colonna Data** contenuto in ogni set di dati rappresenta la data di aggiornamento dei dati nell'app modello.

### <a name="how-is-an-active-user-defined"></a>Come viene definito un utente attivo?

La definizione di utente attivo corrisponde alla definizione [dell'utente attivo](../activity-reports/active-users.md) nei report attività.

### <a name="what-sharepoint-site-collections-are-included-in-the-sharepoint-reports"></a>Quali raccolte siti di SharePoint sono incluse nei report di SharePoint?

La versione corrente dell'app modello include attività di file SharePoint siti del team e SharePoint di gruppo.

### <a name="which-groups-are-included-in-the-microsoft-365-groups-usage-report"></a>Quali gruppi sono inclusi nel report di utilizzo Microsoft 365 gruppi?

La versione corrente dell'app modello include l'uso da Outlook gruppi, Yammer gruppi e SharePoint gruppi. Non include gruppi correlati a Microsoft Teams o Planner.

### <a name="when-will-an-updated-version-of-the-template-app-become-available"></a>Quando sarà disponibile una versione aggiornata dell'app modello?

Le modifiche principali apportate all'app modello verranno rilasciate due volte all'anno, che possono includere nuovi report o nuovi dati. Le modifiche minori apportate ai report possono essere rilasciate con maggiore frequenza.

### <a name="is-it-possible-to-integrate-the-data-from-the-template-app-into-existing-solutions"></a>È possibile integrare i dati dall'app modello alle soluzioni esistenti? 

I dati nell'app modello possono essere recuperati tramite le API Microsoft 365 (in anteprima). Quando vengono spediti in produzione, verranno uniti all'interno delle API [di report di Microsoft Graph](https://go.microsoft.com/fwlink/p/?linkid=848843). 

### <a name="are-there-plans-to-expand-the-template-app-to-show-usage-data-from-other-microsoft-products"></a>Sono in programma di espandere l'app modello per mostrare i dati di utilizzo di altri prodotti Microsoft?

Questo è considerato per miglioramenti futuri. Controlla la roadmap [Microsoft 365 per](https://www.microsoft.com/microsoft-365/roadmap) gli aggiornamenti.

### <a name="how-can-i-pivot-by-company-information-in-active-directory"></a>Come è possibile filtrare le informazioni aziendali in Active Directory?

Le informazioni sulla società sono incluse in uno dei campi di Active Directory nell'app modello e possono essere visualizzate come filtro predefinito nei **report attività utente prodotto.** È disponibile come colonna nella **tabella UserState.**

### <a name="is-it-possible-to-bring-in-additional-fields-from-active-directory"></a>È possibile inserire altri campi da Active Directory?

È possibile eseguire ulteriori personalizzazioni su questi dati connettendosi alle API di report di [Microsoft Graph](https://go.microsoft.com/fwlink/p/?linkid=848843) per estrarre campi aggiuntivi da Azure Active Directory e partecipare al set di dati. 

### <a name="is-it-possible-to-aggregate-the-information-in-the-template-app-across-multiple-subscriptions"></a>È possibile aggregare le informazioni nell'app modello tra più sottoscrizioni?

Al momento, l'app modello è per una singola sottoscrizione, in quanto è associata alle credenziali usate per connettersi inizialmente ad esso.

### <a name="is-it-possible-to-see-usage-by-plan-ie-e1-e3"></a>È possibile visualizzare l'utilizzo per piano (ad esempio E1, E3)?

Nell'app modello l'utilizzo è rappresentato a livello di prodotto. Vengono forniti dati sulle varie sottoscrizioni assegnate agli utenti, tuttavia non è possibile correlare le attività degli utenti alla sottoscrizione assegnata all'utente.

### <a name="is-it-possible-to-integrate-other-data-sets-into-the-template-app"></a>È possibile integrare altri set di dati nell'app modello?

Puoi usare Power BI Desktop per connetterti alle API di Microsoft 365 (in anteprima) per aggiungere origini dati aggiuntive da combinare con i dati dell'app modello.

Per ulteriori informazioni, vedere [Il documento Personalizza](customize-reports.md).

### <a name="is-it-possible-to-see-the-top-users-reports-for-a-specific-timeframe"></a>È possibile visualizzare i report "Utenti principali" per un intervallo di tempo specifico?

Tutti i report a livello utente presentano dati aggregati per il mese precedente.

### <a name="will-the-template-app-be-localized"></a>L'app modello verrà localizzata? 

Questo non è attualmente nella roadmap.

### <a name="i-have-a-specific-question-about-the-data-im-seeing-for-my-organization-who-can-i-reach-out-to"></a>In caso di domande specifiche sui dati visualizzati per l'organizzazione, a chi è possibile rivolgersi?

Puoi usare il pulsante feedback nella pagina panoramica dell'attività [](../../business-video/get-help-support.md) dell'interfaccia di amministrazione oppure puoi aprire un caso di supporto per ottenere assistenza con l'app modello. 

### <a name="how-can-partners-access-the-data"></a>Come possono fare i partner ad accedere ai dati?

Se un partner ha delegato i diritti di amministratore, può connettersi all'app modello per conto del cliente.

### <a name="can-i-hide-identifiable-information-such-as-user-group-and-site-names-in-reports"></a>È possibile nascondere le informazioni personali, come i nomi di utenti, gruppi e siti, nei report?

Sì, vedere [Rendere anonimi i dati raccolti.](enable-usage-analytics.md#make-the-collected-data-anonymous)
