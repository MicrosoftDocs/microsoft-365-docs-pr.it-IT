## <a name="understanding-assessment"></a><span data-ttu-id="581c2-106">Informazioni sulla valutazione</span><span class="sxs-lookup"><span data-stu-id="581c2-106">Understanding assessment</span></span>

<span data-ttu-id="581c2-107">In Valutazione, l'esperto esamina un insieme casuale di almeno 500 file, che vengono utilizzati per determinare la ricchezza dei problemi e per produrre statistiche che riflettono i risultati della formazione.</span><span class="sxs-lookup"><span data-stu-id="581c2-107">In Assessment, the expert reviews a random set of at least 500 files, which are used to determine the richness of the issues and to produce statistics that reflect the training results.</span></span> <span data-ttu-id="581c2-108">La valutazione ha esito positivo quando vengono trovati file pertinenti sufficienti per raggiungere un livello statistico che consente di Advanced eDiscovery Pertinenza per fornire statistiche accurate e determinare in modo efficace il punto di stabilizzazione nel processo di formazione.</span><span class="sxs-lookup"><span data-stu-id="581c2-108">Assessment is successful when enough relevant files are found to reach a statistical level that will help Advanced eDiscovery Relevance to provide accurate statistics and to effectively determine the stabilization point in the training process.</span></span> 
  
<span data-ttu-id="581c2-109">Maggiore è il numero di file rilevanti nel set di valutazioni, più accurate sono le statistiche e l'efficacia dell'algoritmo di stabilità.</span><span class="sxs-lookup"><span data-stu-id="581c2-109">The higher the number of relevant files in the assessment set, the more accurate the statistics and the effectiveness of the stability algorithm.</span></span> <span data-ttu-id="581c2-110">Il numero di file pertinenti all'interno dei file di valutazione dipende dalla ricchezza del problema.</span><span class="sxs-lookup"><span data-stu-id="581c2-110">The number of relevant files within the assessment files depends on the richness of the issue.</span></span> <span data-ttu-id="581c2-111">Richness è la percentuale stimata di file rilevanti nel set rilevante per un problema.</span><span class="sxs-lookup"><span data-stu-id="581c2-111">Richness is the estimated percent of relevant files in the set relevant to an issue.</span></span> <span data-ttu-id="581c2-112">I problemi con una maggiore ricchezza raggiungeranno un numero più elevato di file rilevanti più rapidamente rispetto ai problemi con una minore ricchezza.</span><span class="sxs-lookup"><span data-stu-id="581c2-112">Issues with higher richness will reach a higher number of relevant files more quickly than issues with lower richness.</span></span> <span data-ttu-id="581c2-113">I problemi con una ricchezza estremamente bassa (ad esempio, il 2% o meno) richiederanno un set di valutazione molto grande per raggiungere un numero significativo di file rilevanti.</span><span class="sxs-lookup"><span data-stu-id="581c2-113">Issues with extremely low richness (for example, 2% or less) will require a very large assessment set to reach a significant number of Relevant files.</span></span>
  
<span data-ttu-id="581c2-114">Le statistiche, presentate nelle schede Traccia e Decidi durante l'addestramento e dopo il calcolo batch, includono stime del richiamo per diversi set di revisione.</span><span class="sxs-lookup"><span data-stu-id="581c2-114">The statistics, which are presented in the Track and Decide tabs during training and after Batch calculation, include estimations of recall for different review sets.</span></span> <span data-ttu-id="581c2-115">Nelle statistiche, le stime basate su un set di campioni (in questo caso i file di valutazione) includono il margine di errore e il livello di probabilità di tale margine di errore.</span><span class="sxs-lookup"><span data-stu-id="581c2-115">In statistics, estimations that are based on a sample set (in this case, the assessment files) include the margin of error and the confidence level of that error margin.</span></span> <span data-ttu-id="581c2-116">Ad esempio, il richiamo stimato dell'80% potrebbe avere un margine di errore maggiore o inferiore al 5% con un livello di probabilità del 95%.</span><span class="sxs-lookup"><span data-stu-id="581c2-116">For example, estimated recall of 80% might have a margin of error of plus or minus 5% with a confidence level of 95%.</span></span> <span data-ttu-id="581c2-117">Ciò significa che il richiamo stimato è in realtà del 75%-85% e questa stima ha una probabilità del 95%.</span><span class="sxs-lookup"><span data-stu-id="581c2-117">This means that the estimated recall is actually 75%-85% and this estimation has 95% confidence.</span></span> <span data-ttu-id="581c2-118">Più grande è il set di valutazioni, il margine di errore diventa più piccolo e le statistiche sono più accurate.</span><span class="sxs-lookup"><span data-stu-id="581c2-118">The larger the assessment set, the margin of error becomes smaller and the statistics are more accurate.</span></span> 
  
<span data-ttu-id="581c2-119">Dopo che l'esperto ha verificato un set di valutazione iniziale di 500 file, la rilevanza può determinare il margine di errore corrente dei valori di richiamo.</span><span class="sxs-lookup"><span data-stu-id="581c2-119">After the expert reviews an initial assessment set of 500 files, Relevance can determine the current margin of error of the recall values.</span></span> <span data-ttu-id="581c2-120">La pertinenza consiglia inoltre un margine di errore predefinito da raggiungere per ottimizzare il set di valutazione.</span><span class="sxs-lookup"><span data-stu-id="581c2-120">Relevance will also recommend a default margin of error to reach to optimize the assessment set.</span></span> <span data-ttu-id="581c2-121">Di seguito vengono descritti alcuni esempi:</span><span class="sxs-lookup"><span data-stu-id="581c2-121">Here are some examples:</span></span>
  
- <span data-ttu-id="581c2-122">Se il set di valutazioni ha già restituito un margine di errore maggiore o inferiore al 10%, Pertinenza consiglia di passare alla formazione (non è necessaria alcuna revisione di valutazione aggiuntiva).</span><span class="sxs-lookup"><span data-stu-id="581c2-122">If the assessment set already yielded a margin of error of plus or minus 10%, Relevance will recommend moving on to training (no additional assessment review is needed).</span></span> 

- <span data-ttu-id="581c2-123">Se il set di valutazioni ha restituito un margine di errore maggiore o minore del 13%, la pertinenza potrebbe consigliare la revisione di un altro set di file di valutazione per raggiungere un margine inferiore.</span><span class="sxs-lookup"><span data-stu-id="581c2-123">If the assessment set yielded a margin of error of plus or minus 13%, Relevance might recommend the review of another set of assessment files to reach a smaller margin.</span></span> 

- <span data-ttu-id="581c2-124">Se la ricchezza è estremamente bassa, la pertinenza potrebbe consigliare di interrompere la valutazione anche se il margine di errore è elevato (rendendo le statistiche poco pratiche), perché il set di valutazione necessario per raggiungere un margine di errore utile è troppo grande.</span><span class="sxs-lookup"><span data-stu-id="581c2-124">If richness is extremely low, Relevance might recommend stopping assessment even though the margin of error is large (making statistics impractical), because the assessment set needed to reach a useful margin of error is too large.</span></span>

<span data-ttu-id="581c2-125">Ogni problema ha la propria ricchezza, il margine di errore corrente e, di conseguenza, il numero stimato di file di valutazione aggiuntivi.</span><span class="sxs-lookup"><span data-stu-id="581c2-125">Each issue has its own richness, current margin of error, and as a result, estimated number of additional assessment files.</span></span> <span data-ttu-id="581c2-126">Il set di valutazione successivo viene creato in base al numero massimo di file (fino a 1.000 in un singolo set).</span><span class="sxs-lookup"><span data-stu-id="581c2-126">The next assessment set is created according to the maximum number of files (up to 1,000 in a single set).</span></span>
