# <a name="privileged-access-management-in-microsoft-365"></a>Gestione degli accessi privilegiati in Microsoft 365

Avere accesso permanente da parte di alcuni utenti a informazioni riservate o impostazioni di configurazione di rete critiche in Microsoft Exchange Online è un potenziale percorso per gli account compromessi o le attività di minacce interne. La gestione degli accessi privilegiati consente di proteggere l'organizzazione dalle violazioni e consente di soddisfare le procedure consigliate di conformità limitando l'accesso permanente ai dati sensibili o l'accesso alle impostazioni di configurazione critiche. Anziché gli amministratori hanno accesso costante, vengono implementate regole di accesso just-in-time per le attività che necessitano di autorizzazioni elevate. L'abilitazione della gestione degli accessi con privilegi per Exchange Online in Microsoft 365 consente all'organizzazione di operare senza privilegi permanenti e fornire un livello di difesa contro le vulnerabilità di accesso amministrativo permanenti.

## <a name="configure-privileged-access-management-for-microsoft-365"></a>Configurare la gestione degli accessi con privilegi per Microsoft 365

Per configurare la gestione degli accessi con privilegi per l'organizzazione, eseguire la procedura seguente:

![Passaggi di gestione degli accessi privilegiati della soluzione di rischio Insider](../media/ir-solution-pam-steps.png)

1. Informazioni sulla [gestione degli accessi con privilegi](privileged-access-management-overview.md) in Microsoft 365
2. Creare un [gruppo di responsabili approvazione](privileged-access-management-configuration.md#step-1-create-an-approvers-group)
3. Abilitare [la gestione degli accessi con privilegi](privileged-access-management-configuration.md#step-2-enable-privileged-access)
4. Creare un criterio [di accesso](privileged-access-management-configuration.md#step-3-create-an-access-policy)
5. Inviare/approvare [richieste di accesso con privilegi](privileged-access-management-configuration.md#step-4-submitapprove-privileged-access-requests)
